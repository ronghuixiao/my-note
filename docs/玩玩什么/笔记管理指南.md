## 一、核心原则

你的笔记网站由两部分组成：

| 部分 | 位置 | 作用 |
|------|------|------|
| **内容文件** | `docs/` 目录下的 `.md` 文件 | 笔记的实际内容 |
| **配置文件** | `docs/.vitepress/config.mts` | 控制导航栏、侧边栏、网站标题等 |

**每次增删内容后，必须同步修改配置文件，否则页面上不会显示或会残留已删除的链接。**

## 二、增加一篇笔记

### 步骤 1：创建 Markdown 文件

在 `docs/` 目录下对应的分类文件夹中新建 `.md` 文件。

例如，要新增一篇「Vue」笔记：

```
docs/
└── 前端/
    ├── JavaScript.md
    ├── CSS.md
    └── Vue.md          ← 新建这个文件
```

文件内容示例：

```markdown
# Vue 学习笔记

## 基础概念

在这里写内容...

## 组件化开发

在这里写内容...
```

### 步骤 2：在 config.mts 的 sidebar 中添加链接

打开 `docs/.vitepress/config.mts`，在对应分类的 `items` 中添加一行：

```ts
sidebar: [
  {
    text: '前端知识',
    items: [
      { text: 'JavaScript', link: '/前端/JavaScript' },
      { text: 'CSS', link: '/前端/CSS' },
      { text: 'Vue', link: '/前端/Vue' },        // ← 新增
    ]
  },
]
```

> **link 的写法**：`/分类文件夹名/文件名`（不带 `.md` 后缀）

### 步骤 3：验证

运行 `npm run dev`，检查：
- 左侧边栏是否出现新笔记的链接
- 点击链接后页面内容是否正确

## 三、删除一篇笔记

### 步骤 1：删除 Markdown 文件

直接删除对应的 `.md` 文件。

例如删除 CSS 笔记：

```
删除：docs/前端/CSS.md
```

### 步骤 2：在 config.mts 的 sidebar 中移除链接

从 `items` 数组中删掉对应的那一行：

```diff
  sidebar: [
    {
      text: '前端知识',
      items: [
        { text: 'JavaScript', link: '/前端/JavaScript' },
-       { text: 'CSS', link: '/前端/CSS' },
      ]
    },
  ]
```

> ⚠️ **如果不删配置，侧边栏会显示一个点击后 404 的死链接。**

## 四、修改笔记内容

直接编辑 `docs/` 下的 `.md` 文件即可，**不需要修改 config.mts**。

支持的 Markdown 功能：

```markdown
# 一级标题
## 二级标题

**加粗** *斜体* ~~删除线~~

- 无序列表
1. 有序列表

> 引用

`行内代码`

[链接文字](https://example.com)

![图片描述](./image.png)
```

### 插入图片

将图片放在 `docs/` 下的任意位置（推荐建一个 `assets` 文件夹），然后在 Markdown 中引用：

```markdown
![截图](/前端/assets/screenshot.png)
```

或者直接粘贴到编辑器中，VitePress 会自动处理路径。

## 五、增加一个分类（侧边栏分组）

### 步骤 1：创建分类文件夹

```
docs/
├── 前端/
├── 后端/
└── 工具/          ← 新建分类文件夹
    └── Git.md     ← 分类下的笔记
```

### 步骤 2：在 sidebar 中添加分组

```ts
sidebar: [
  {
    text: '前端知识',
    items: [
      { text: 'JavaScript', link: '/前端/JavaScript' },
      { text: 'CSS', link: '/前端/CSS' },
    ]
  },
  {
    text: '后端知识',
    items: [
      { text: 'Python', link: '/后端/Python' },
    ]
  },
  {
    text: '开发工具',           // ← 新增分类
    items: [
      { text: 'Git', link: '/工具/Git' },
    ]
  }
]
```

## 六、删除一个分类

### 步骤 1：删除整个文件夹

直接删除 `docs/` 下对应的文件夹（会连同里面所有笔记一起删除）。

### 步骤 2：在 sidebar 中移除整个分组

```diff
  sidebar: [
    {
      text: '前端知识',
      items: [...]
    },
-   {
-     text: '后端知识',
-     items: [
-       { text: 'Python', link: '/后端/Python' },
-     ]
-   },
  ]
```

## 七、调整导航栏（顶部导航）

导航栏在 `config.mts` 的 `nav` 数组中配置，与 sidebar 独立。

### 新增顶部导航项

```ts
nav: [
  { text: '首页', link: '/' },
  { text: '前端', link: '/前端/JavaScript' },   // ← 新增
  { text: '后端', link: '/后端/Python' },        // ← 新增
],
```

### 添加下拉菜单导航

```ts
nav: [
  { text: '首页', link: '/' },
  {
    text: '前端',
    items: [
      { text: 'JavaScript', link: '/前端/JavaScript' },
      { text: 'CSS', link: '/前端/CSS' },
    ]
  },
  {
    text: '后端',
    items: [
      { text: 'Python', link: '/后端/Python' },
    ]
  }
],
```

### 移除导航项

直接删掉 `nav` 数组中对应的对象即可。

## 八、新增一个首页功能卡片

首页内容在 `docs/index.md` 中配置，使用 VitePress 的 Front Matter 语法。

### 新增一个卡片

在 `features` 数组中添加一项：

```yaml
---
layout: home

hero:
  name: "我的笔记"
  text: "记录学习与思考"
  tagline: 个人知识库，持续更新中
  actions:
    - theme: brand
      text: 开始阅读
      link: /前端/JavaScript

features:
  - title: 前端知识
    details: JavaScript、CSS、HTML 等前端技术的学习笔记
  - title: 后端知识
    details: Python、数据库等后端技术的学习笔记
  - title: 开发工具                # ← 新增卡片
    details: Git、VS Code 等开发工具的使用技巧
---
```

### 删除一个卡片

从 `features` 数组中移除对应的 `- title` 块即可。

## 九、文件路径速查表

```
d:\my-note\
├── package.json                         ← 项目配置（一般不用改）
├── .gitignore                           ← Git 忽略规则（一般不用改）
├── 笔记管理指南.md                       ← 本文档
├── .github/
│   └── workflows/
│       └── deploy.yml                   ← GitHub Actions 自动部署（一般不用改）
└── docs/
    ├── index.md                         ← 首页内容
    └── .vitepress/
        ├── config.mts                   ← ⭐ 核心配置（导航、侧边栏、标题）
        ├── cache/                       ← 构建缓存（自动生成，不用管）
        └── dist/                        ← 构建产物（自动生成，不用管）
```

### 常改的文件

| 场景 | 改哪个文件 |
|------|-----------|
| 写/改笔记内容 | `docs/对应分类/笔记.md` |
| 增删笔记链接 | `docs/.vitepress/config.mts` → `sidebar` |
| 改顶部导航 | `docs/.vitepress/config.mts` → `nav` |
| 改首页内容 | `docs/index.md` |
| 改网站标题/描述 | `docs/.vitepress/config.mts` → `title` / `description` |
| 改 GitHub 链接 | `docs/.vitepress/config.mts` → `socialLinks` |

## 十、常见问题

### Q：新增笔记后页面上看不到？

**A：** 检查 `config.mts` 的 `sidebar` 是否添加了对应的 link。

### Q：删除笔记后侧边栏还有链接，点进去 404？

**A：** 忘记从 `sidebar` 中移除对应的行了。

### Q：修改了笔记内容但线上没更新？

**A：** 代码推送到 `main` 分支后 GitHub Actions 会自动部署，检查是否推送成功。进入仓库的 Actions 页面查看构建状态。

### Q：如何给笔记添加标签或分类？

**A：** VitePress 支持在 Markdown 文件顶部添加 Front Matter：

```markdown
---
title: JavaScript 基础
tags:
  - 前端
  - JavaScript
---
```

### Q：如何在笔记之间添加链接？

**A：** 使用 Markdown 链接，指向目标文件的路径（不带 `.md` 后缀）：

```markdown
参见 [CSS 学习笔记](/前端/CSS)
```

### Q：如何调整侧边栏的展开/折叠？

**A：** 在 `config.mts` 中给侧边栏分组设置 `collapsed`：

```ts
sidebar: [
  {
    text: '前端知识',
    collapsed: false,     // 默认展开
    items: [...]
  },
  {
    text: '后端知识',
    collapsed: true,      // 默认折叠
    items: [...]
  }
]
```
