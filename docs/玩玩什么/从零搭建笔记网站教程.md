## 第一步：环境准备

你需要安装以下工具：

| 工具 | 用途 | 安装方式 |
|------|------|---------|
| **Node.js** (v18+) | 运行环境 | 去 [nodejs.org](https://nodejs.org) 下载 LTS 版本 |
| **npm** | 包管理器 | 安装 Node.js 后自带 |
| **Git** | 版本管理 | 去 [git-scm.com](https://git-scm.com) 下载 |
| **VS Code** | 代码编辑器 | 去 [code.visualstudio.com](https://code.visualstudio.com) 下载 |

验证安装：

```bash
node -v      # 应显示 v18.x.x 或更高
npm -v       # 应显示 9.x.x 或更高
git --version
```

## 第二步：初始化项目

### 2.1 创建项目文件夹

```bash
mkdir my-note
cd my-note
```

### 2.2 初始化 npm

```bash
npm init -y
```

这会生成一个 `package.json` 文件。

### 2.3 安装 VitePress

```bash
npm install -D vitepress
```

### 2.4 添加便捷脚本

编辑 `package.json`，把 `scripts` 部分改成：

```json
{
  "scripts": {
    "dev": "vitepress dev docs",
    "build": "vitepress build docs",
    "preview": "vitepress preview docs"
  }
}
```

三个命令的作用：

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `npm run dev` | 启动本地预览服务器 | 日常写笔记时实时预览 |
| `npm run build` | 构建静态网站文件 | 部署前的构建 |
| `npm run preview` | 预览构建后的效果 | 检查构建结果是否正确 |

### 2.5 创建目录结构

```bash
mkdir docs/.vitepress
```

此时项目结构：

```
my-note/
├── package.json
├── package-lock.json
├── node_modules/
└── docs/
    └── .vitepress/
```

### 2.6 初始化 Git

```bash
git init
```

## 第三步：基础配置

创建配置文件 `docs/.vitepress/config.mts`：

```ts
import { defineConfig } from 'vitepress'

export default defineConfig({
  // 网站标题（显示在浏览器标签页）
  title: '我的笔记',

  // 网站描述（SEO 用）
  description: '记录我的学习和思考',

  // 如果部署到 https://用户名.github.io/仓库名/，需要设置 base
  // 如果部署到 https://用户名.github.io/，则不需要这行
  base: '/你的仓库名/',

  // 主题配置
  themeConfig: {
    // ---- 导航栏（顶部） ----
    nav: [
      { text: '首页', link: '/' },
      { text: '前端', link: '/前端/JavaScript' },
      { text: '后端', link: '/后端/Python' },
    ],

    // ---- 侧边栏（左侧） ----
    sidebar: [
      {
        text: '前端知识',
        items: [
          { text: 'JavaScript', link: '/前端/JavaScript' },
          { text: 'CSS', link: '/前端/CSS' },
        ]
      },
      {
        text: '后端知识',
        items: [
          { text: 'Python', link: '/后端/Python' },
        ]
      }
    ],

    // ---- 社交链接（右上角图标） ----
    socialLinks: [
      { icon: 'github', link: 'https://github.com/你的用户名' }
    ],

    // ---- 页脚 ----
    footer: {
      message: '基于 VitePress 构建',
      copyright: '© 2024 我的笔记'
    }
  }
})
```

验证配置是否正确：

```bash
npm run dev
```

浏览器打开 `http://localhost:5173`，能看到页面就说明成功了。

## 第四步：创建内容结构

### 4.1 创建首页

创建 `docs/index.md`：

```markdown
---
layout: home

hero:
  name: "我的笔记"
  text: "记录学习与思考"
  tagline: 个人知识库，持续更新中
  actions:
    - theme: brand
      text: 开始阅读
      link: /前端/JavaScript

  - title: 前端知识
    details: JavaScript、CSS 等前端技术笔记
  - title: 后端知识
    details: Python、数据库等后端技术笔记

根据 `config.mts` 中 sidebar 的配置，创建对应的文件：

```bash
mkdir -p docs/前端 docs/后端
```

然后创建 Markdown 文件：

```bash
touch docs/前端/JavaScript.md
touch docs/前端/CSS.md
touch docs/后端/Python.md
```

my-note/

## 第五步：主题美化

### 5.1 使用内置深色模式

VitePress 自带深色模式支持，在 `config.mts` 中添加：

```ts
export default defineConfig({
  // ...其他配置

  themeConfig: {
    // 启用深色模式切换按钮（右上角太阳/月亮图标）
    // VitePress 默认已启用，无需额外配置

    // 如果想默认使用深色模式：
    // appearance: 'dark',

    // ...其他配置
  }
})
```

### 5.2 自定义 CSS 样式

创建自定义样式文件 `docs/.vitepress/theme/custom.css`：

```css
/* ============================================
   全局颜色变量覆盖

### 5.3 创建自定义主题入口

创建 `docs/.vitepress/theme/index.ts`：

```ts
import DefaultTheme from 'vitepress/theme'
import './custom.css'

export default {
  extends: DefaultTheme,
}
```

### 5.4 应用自定义主题

在 `config.mts` 中添加 `theme` 配置：

```ts
export default defineConfig({
  // ...其他配置

  // 指向自定义主题
  theme: './.vitepress/theme/index.ts',
})
```

## 第六步：插件增强

VitePress 支持通过插件扩展功能。以下是一些实用的插件：

### 6.1 搜索功能（内置）

VitePress 已内置本地搜索，无需安装插件。在 `config.mts` 中启用：

```ts
export default defineConfig({
  themeConfig: {
    // 本地搜索
    search: {
      provider: 'local',
      options: {
        translations: {
          button: {
            buttonText: '搜索文档',
            buttonAriaLabel: '搜索'
          },
          modal: {
            noResultsText: '无法找到相关结果',
            resetButtonTitle: '清除查询条件',
            footer: {
              selectText: '选择',
              navigateText: '切换'
            }
          }
        }
      }
    }
  }
})
```

效果：页面右上角出现搜索框，支持全文搜索，中文友好。

### 6.2 Markdown 增强（markdown-it 插件）

VitePress 底层使用 markdown-it，可以直接使用其插件来增强 Markdown 渲染。

#### 6.2.1 安装

```bash
npm install -D markdown-it-mark          # 高亮文字 ==高亮==
npm install -D markdown-it-footnote      # 脚注支持 [^1]
```

#### 6.2.2 配置

在 `config.mts` 中添加：

```ts
import footnote from 'markdown-it-footnote'
import mark from 'markdown-it-mark'

export default defineConfig({
  markdown: {
    config: (md) => {
      md.use(footnote)
      md.use(mark)
    }
  }
})
```

#### 6.2.3 使用

在 Markdown 中：

```markdown
这段文字有==高亮==效果。

这是脚注的引用[^1]。

[^1]: 这是脚注的内容。
```

### 6.3 代码高亮增强

VitePress 使用 Shiki 进行代码高亮，支持多种语言。在 `config.mts` 中配置：

```ts
export default defineConfig({
  markdown: {
    // 行号显示
    lineNumbers: true,

    // 代码块标题（显示文件名）
    // 使用 ```js filename="xxx.js" 语法

    // 自定义语言支持
    // shikiSetup: async (highlighter) => { ... }
  }
})
```

使用效果：

````markdown
```js filename="utils.js"
// 代码块会显示文件名
function greet(name) {
  console.log(`Hello, ${name}!`)
}
```
````

### 6.4 图片缩放插件（vitepress-plugin-group-icons）

```bash
npm install -D vitepress-plugin-group-icons
```

在 `config.mts` 中配置：

```ts
import { groupIconsPlugin } from 'vitepress-plugin-group-icons'

export default defineConfig({
  markdown: {
    config: (md) => {
      md.use(groupIconsPlugin)
    }
  }
})
```

## 第八步：部署到 GitHub Pages

### 8.1 创建 GitHub 仓库

1. 登录 [github.com](https://github.com)
2. 点击右上角 "+" → "New repository"
3. 仓库名填 `你的仓库名`（如 `my-note`）
4. 选择 Public
5. 点击 Create

### 8.2 推送代码

```bash
git init
git add .
git commit -m "init vitepress site"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

### 8.3 创建自动部署配置

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy VitePress to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Install dependencies
        run: npm ci
      - name: Build with VitePress
        run: npm run build
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 8.4 启用 GitHub Pages

1. 进入仓库 → Settings → Pages
2. Source 选择 **GitHub Actions**（不是 "Deploy from a branch"）
3. 保存

### 8.5 完成

推送代码后，网站会自动部署。访问地址：

```
https://你的用户名.github.io/你的仓库名/
```

## 第九步：进阶优化

### 9.1 SEO 优化

在 `config.mts` 中添加：

```ts
export default defineConfig({
  // Meta 标签
  head: [
    ['meta', { name: 'author', content: '你的名字' }],
    ['meta', { name: 'keywords', content: '笔记,前端,后端,学习' }],
    ['meta', { name: 'description', content: '个人学习笔记网站' }],

    // Open Graph（社交媒体分享时显示）
    ['meta', { property: 'og:title', content: '我的笔记' }],
    ['meta', { property: 'og:description', content: '个人学习笔记' }],
    ['meta', { property: 'og:image', content: '/og-image.png' }],
  ],
})
```

### 9.2 最后更新时间

VitePress 可以自动显示页面的最后更新时间：

```ts
export default defineConfig({
  lastUpdated: true,
})
```

### 9.3 目录（TOC）

```ts
export default defineConfig({
  markdown: {
    // 每个页面右侧自动生成目录
    // VitePress 默认已有此功能
  },
  themeConfig: {
    // 目录标题
    outline: {
      level: [2, 3],       // 显示 h2-h3 级别的标题
      label: '页面导航'
    }
  }
})
```

### 9.4 上一篇/下一篇导航

在 `config.mts` 中配置：

```ts
export default defineConfig({
  themeConfig: {
    docFooter: {
      prev: '上一篇',
      next: '下一篇'
    }
  }
})
```

VitePress 会根据 sidebar 的顺序自动生成上下页链接。

### 9.5 自定义 404 页面

创建 `docs/404.md`：

```markdown
---
layout: page
---

# 404 - 页面不存在

抱歉，你访问的页面不存在。

[返回首页](/)
```

### 9.6 离线 PWA 支持

安装插件：

```bash
npm install -D vitepress-plugin-pwa
```

在 `config.mts` 中配置：

```ts
import { PWAPlugin } from 'vitepress-plugin-pwa'

export default defineConfig({
  plugins: [
    PWAPlugin({
      registerType: 'autoUpdate',
      manifest: {
        name: '我的笔记',
        short_name: '笔记',
        theme_color: '#3451b2',
      }
    })
  ]
})
```

效果：用户可以将网站"安装"到桌面，支持离线访问。
